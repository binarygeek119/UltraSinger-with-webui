System

Processing

yt-dlp cookies

Used for YouTube and other yt-dlp sources when videos are age-restricted, members-only, or show bot-check errors. Paste the full contents of a Netscape-format cookies.txt (e.g. from yt-dlp --cookies-from-browser firefox or an exporter extension), or set a file path below. Saved text is written to data/cookies.txt and passed to UltraSinger as --cookiefile.

Saving pasted cookies updates this path to data/cookies.txt automatically.

Default job

Used only when queueing with “Use YouTube metadata only”. A frame is captured from the downloaded video at this percentage of total duration and saved as * [BG].jpg (default: 30).

When YARG mode is on, per-job and bulk ZIP downloads use the same flat names as YARG folder export (guitar.*, background.*, notes.txt, album.*) under each song folder; .mid is omitted from those ZIPs. When off, ZIPs keep normal UltraSinger paths (and optional album rename when “omit Instrumental/Vocals” is checked).

Downloads

YARG and UltraStar export toggles and folder paths are also written to export_folders.json in your data directory whenever settings are saved. If that file is newer than webui_config.json, those four fields are reloaded from it on startup (handy for hand-editing paths).

Applies to per-job ZIP and “download all completed”. Song folder files on disk are unchanged. When enabled, the cover image in those ZIPs is named album.<ext> inside each song folder (same pick as YARG folder export). When off, original filenames are kept in the ZIP.

When enabled and a path is set, each successful job writes a flat folder <folder>/<Artist - Title>/ with fixed names (same inclusion rules as the YARG-style ZIP: respects Instrumental/Vocals omit above, no .mid): guitar.<ext> (instrumental stem if present, else main mix audio), background.<ext> (video), notes.txt (UltraStar txt), album.<ext> (cover image).

When enabled with a path, copies the full song output (.txt, .mid, [Instrumental] / [Vocals] audio, cover, video, etc.) into <folder>/<Artist - Title>/. For these runs, YARG mode is not applied so UltraSinger still writes .mid and stem tracks (your “YARG mode” checkbox is ignored only while this export is active and the path is set).
